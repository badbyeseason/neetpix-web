| # | 偏好 | 说明 |

| # | 排除方向 | 理由 |

| 3 | **体系化电商** | 中国电商生态已极度成熟、体系完整 |
| 4 | **纯 AI 聊天包装** | API 封装无壁垒，容易被复制 |

## 第三部分：方法论框架（已开发）

在探索过程中，以下方法论框架被开发和使用过：

### 方法论 1：4 框架交叉法

四个维度交叉过滤候选品类：

1. **A 问题空间扫描** — 信息管理 / 内容创作 / 自动化处理 / 知识学习 / 消费决策 / 隐私清理
2. **B 市场验证模式** — 被独立开发者验证过的产品品类（Product Hunt、AppSumo 等平台反复出现的品类）
3. **C AI 护城河评估** — AI 能成为真正壁垒的能力类型（OCR、语义搜索、结构理解 > API 包装）
4. **D 分发匹配** — 在受众为零的前提下，产品的 SEO/搜索发现潜力

### 方法论 2：5 种用户切割法

用于重新定义用户分析视角：

| 切割法 | 核心问题 | 适用场景 |
|--------|---------|---------|
| A. JTBD（待完成的任务） | 人们在做什么事？哪里卡住了？ | 从具体场景出发找机会 |
| B. 供需错配 | 中国 vs 全球市场之间有什么结构性缺口？ | 发挥双语+双文化优势 |
| C. AI 时代新痛点 | AI 普及后产生了什么新的麻烦？ | 做"AI 时代的真问题" |
| D. 工作流形态 | 不同行业但有相同工作流的人群 | 跨行业的水平机会 |
| E. 行为迁移 | 什么正在变得比 3 年前更难/更频繁？ | 趋势驱动的方向 |

### 方法论 3：双 Agent 辩论法

流程：
1. **Agent 1** — 从第一性原理出发，提出产品方向方案
2. **Agent 2** — 对抗式审查，通过持续追问迫使 Agent 1 修正/改进方案
3. 循环直到方案经得起攻击

此方法是用户主动提出的推进方式。

### 方法论 4：第一性原理推导

从不可变事实出发推导产品方向：

| 第一性原理 | 含义 |
|-----------|------|
| 数字资产膨胀，检索能力下降 | 人产生越来越多数据，但找到数据的能力没有同比提升 |
| AI 与用户上下文隔离 | 当前 AI 工具不了解用户的私有数据 |
| 双语不是翻译问题，是组织问题 | 双语用户的信息天然分成两个孤岛 |
| 高频无意识重复值得自动化 | 最好的产品消除用户意识不到的摩擦 |
