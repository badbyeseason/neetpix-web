## 品类一：网页数据提取 / 抓取（Web Scraper）

**为什么选这个品类：** 需求强且持续增长（数据驱动决策、竞品监控、价格追踪等场景），付费意愿高（B 端用户为主），现有产品普遍体验差。

### 1. Web Scraper (Free Web Scraping)
- **用户数：** ~1,000,000+
- **评分：** ~3.8/5
- **差评核心痛点：**
  - 爬取动态加载内容（JavaScript 渲染页面）经常失败
  - 选择器设置门槛高，非技术用户完全无法上手
  - 大量爬取时极其缓慢，无法并行
  - 爬取结果格式混乱，导出 CSV 经常错位
  - 没有定时自动爬取功能（需手动触发）
  - 网站结构变化后，爬虫直接报废，没有智能修复
- **机会点：** 做一个“零代码 + 智能”的数据提取工具，重点解决动态页面和自动调度

### 2. Data Miner
- **用户数：** ~500,000+
- **评分：** ~3.7/5
- **差评核心痛点：**
  - 免费版极度限制（每月 500 页），付费版价格过高（/月）
  - 公共模板经常失效，没人维护
  - 分页爬取经常卡住或跳过页面
  - 数据清洗能力几乎没有，导出后还要手动整理
  - 客服响应极慢
- **机会点：** 更合理的免费/付费梯度 + 自动维护模板 + 内置数据清洗

### 3. Instant Data Scraper
- **用户数：** ~700,000+
- **评分：** ~4.0/5（相对较好，但仍有明确痛点）
- **差评核心痛点：**
  - 自动检测表格/列表经常误判，提取到无关数据
  - 无法处理需要登录的页面
  - 没有批量 URL 爬取功能（只能一页一页来）
  - 导出格式有限，不支持直接导出到 Google Sheets
  - 没有增量爬取能力（每次都从头开始）
- **机会点：** 智能检测优化 + 支持 Google Sheets 直出 + 增量更新

**品类总结：** 这个品位的用户痛点非常集中——“我想提取数据，但现有工具要么太难用、要么太贵、要么不够智能”。一个“更智能、更简单、价格更合理”的数据提取工具是明确的机会。

## 品类二：标签 / 会话管理（Tab Manager）

**为什么选这个品类：** 高频刚需（几乎每个知识工作者都有“标签页太多”的痛苦），使用场景明确，功能边界清晰，适合做小而专的工具。

### 1. Workona
- **用户数：** ~300,000+
- **评分：** ~3.5/5
- **差评核心痛点：**
  - 强制登录账号，数据存在云端，隐私担忧严重
  - 免费版限制太多（工作区数量、标签数量都有限制）
  - 付费价格高（/月），对于“管标签”这个需求来说太贵
  - 打开保存的会话时，所有标签同时加载，严重拖慢浏览器
  - 偶尔丢失标签数据，用户投诉频繁
  - 过度复杂——用户只想管标签，不想要项目管理功能
- **机会点：** 纯本地存储（隐私安全）+ 轻量级 + 按需加载标签

### 2. Tab Wrangler
- **用户数：** ~200,000+
- **评分：** ~3.8/5
- **差评核心痛点：**
  - 自动关闭标签的逻辑太激进，经常关掉正在用的标签
  - 白名单设置麻烦，没有智能学习机制
  - 恢复被关闭的标签后，表单数据丢失
  - 没有分组/项目维度的标签管理
  - 界面简陋，无法快速浏览和搜索已关闭的标签
- **机会点：** 智能休眠（而非关闭）+ 学习用户习惯 + 保留表单状态

### 3. The Great Suspender（原版已下架，用户仍在寻找替代品）
- **用户数：** ~2,000,000+（下架前）
- **评分：** N/A（已下架）
- **差评核心痛点（来自替代品和论坛讨论）：**
  - 原版被 Google 下架（含恶意代码），用户对类似工具的信任度下降
  - 后续的 fork 版本更新不及时，不兼容新 Chrome 版本
  - 休眠后的标签恢复慢，有时需要重新加载整个页面
  - 没有白名单/黑名单的精细控制
  - 不支持按窗口/项目维度管理
- **机会点：** 做一个可信的、开源的、功能完善的标签休眠管理器，填补这个巨大缺口

**品类总结：** 核心痛点是“我的标签太多太乱，但现有工具要么太重、要么太激进、要么不可信”。一个轻量、智能、尊重隐私的标签管理工具有清晰的市场空间。

## 品类三：截图 & 标注工具（Screenshot & Annotation）

**为什么选这个品类：** 日常高频需求（工作沟通、Bug 报告、设计反馈、社交分享），用户群广泛，现有头部产品正在“变重变贵”，给了轻量替代品空间。

### 1. Awesome Screenshot & Screen Recorder
- **用户数：** ~2,000,000+
- **评分：** ~3.6/5
- **差评核心痛点：**
  - 功能越加越多（录屏、GIF、视频编辑），启动越来越慢
  - 免费版限制越来越大，几乎强制升级到 Pro（/月）
  - 截图标注功能不如以前流畅，经常卡顿
  - 隐私担忧——截图会上传到他们的服务器
  - 录屏功能占用大量资源，导致浏览器卡顿
  - 老用户抱怨产品方向偏了（从截图工具变成了录屏+协作套件）
- **机会点：** 做“纯截图+标注”，不塞无关功能，轻量快速，本地优先

### 2. GoFullPage - Full Page Screenshot
- **用户数：** ~1,500,000+
- **评分：** ~3.9/5
- **差评核心痛点：**
  - 长页面截图经常拼接错位（特别是有懒加载的页面）
  - 截图分辨率低，放大后模糊
  - 不支持选择区域截图（只能截整页）
  - 没有 PDF 导出选项
  - 对含 iframe/嵌套内容的页面截图失败率高
  - 免费版有水印
- **机会点：** 更智能的页面拼接算法 + 多种截图模式 + 高清输出

### 3. Lightshot Screenshot
- **用户数：** ~1,000,000+
- **评分：** ~3.7/5
- **差评核心痛点：**
  - 截图后上传到他们的服务器，隐私风险大
  - 标注工具过于简单（只有基础箭头和文字）
  - 不支持滚动截图
  - 快捷键经常与其他扩展冲突
  - 搜索功能（以图搜图）经常不可用
  - 更新频率极低，Bug 长期不修
- **机会点：** 本地优先 + 丰富标注 + 滚动截图 + 不强制上传

**品类总结：** 核心痛点是“我只想快速截图和标注，但现有工具要么越来越臃肿、要么侵犯隐私、要么截图质量差”。一个专注于“快速截图+专业标注”、尊重隐私、不搞捆绑的纯工具，有明确的差异化空间。

## 机会优先级排序

| 优先级 | 品类 | 理由 |
|--------|------|------|
| ★★★ | 网页数据提取 | 付费意愿最强，B 端用户为主，ARPU 高，市场在增长 |
| ★★★ | 截图 & 标注 | 用户基数最大，头部产品正在“变重变贵”让出空间，开发难度适中 |
| ★★ | 标签/会话管理 | 痛点真实但天花板偏低，更适合作为第一个练手项目 |

## 交叉机会（跨品类组合）

值得关注的是“截图 + 数据提取”的组合场景：
- 截图后自动识别并提取图片中的文字/表格（OCR + 结构化）
- 从截图中提取特定数据（价格、联系方式等）
- 这本质上是把“截图工具”和“数据提取”合并，可以做差异化定价
